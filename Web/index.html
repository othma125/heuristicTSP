Method

The approach

Memetic optimization for the symmetric Traveling Salesman Problem. Given n cities and symmetric distances, the solver searches for a minimum-length Hamiltonian cycle — a closed tour visiting every city exactly once — by hybridizing a genetic algorithm with an embedded local search.

Permutation representation

Each individual is a permutation of the cities. The population is initialized with random permutations that are immediately improved by local search, and evolves under tournament selection (size 5), order-based crossover (rate 0.9) and 2-opt mutation (rate 0.1).

Embedded local search

Every offspring is refined in place with 2-opt plus swap and left/right shift moves, and a stagnation breaker escapes local optima. This makes the algorithm memetic: the genetic operators explore, while local search intensifies each candidate before it competes.

Parallelism & stopping

Offspring are built and evaluated on a fixed thread pool sized to the available CPU cores. Distances use a dense matrix for small instances and a concurrent cache for larger ones. The run stops on a dynamic, size-aware time budget (or when you press Stop). Built in Java 23.
